# 项目开发日志

## 2026-05-26 更新

### 已完成功能

#### 后端（FastAPI）
1. **考试模块** - 新增文字版雅思口语模拟考试
   - `POST /exam/start` - 开始考试，选择题目创建会话
   - `POST /exam/answer` - 提交回答，获取追问
   - `POST /exam/finish` - 结束考试，获取评分报告
   
2. **数据库模型**
   - `ExamSession` - 考试会话表（存储会话信息、评分、反馈）
   - `ExamMessage` - 考试问答记录表（存储每轮对话）

3. **业务逻辑**（app/services/exam_service.py）
   - 规则生成追问（5个候选问题随机选择）
   - 随机生成模拟评分（5.5-7.5 分）
   - 最多支持 3 轮问答
   - 未接入真实大模型

#### 前端（React + Vite）
1. **项目结构**
   - 使用 React 18 + Vite
   - 原生 CSS 样式（无 UI 组件库）
   - 使用 useState 管理页面状态（未使用 React Router）

2. **页面流程**
   - 首页：显示标题和"开始口语练习"按钮
   - 选题页：展示题库列表，点击选择题目
   - 考试页：显示考官问题、用户输入框、对话历史
   - 结果页：展示评分报告（总分、流利度、词汇、语法、反馈、建议）

3. **API 调用**（src/api.js）
   - 统一管理后端接口调用
   - API_BASE_URL: http://localhost:8000

### 当前问题

#### 🔴 CORS 跨域问题（未解决）

**问题描述：**
- 前端（http://localhost:5173）无法访问后端（http://localhost:8000）
- 浏览器报错：`Access to fetch at 'http://localhost:8000/questions' from origin 'http://localhost:5173' has been blocked by CORS policy`
- 后端响应头中缺少 `Access-Control-Allow-Origin` 头

**已尝试的解决方案：**
1. ✅ 在 app/main.py 中添加 CORSMiddleware
2. ✅ 修改 allow_origins 为 `["*"]`（允许所有来源）
3. ✅ 调整中间件顺序（移到路由注册之前/之后）
4. ✅ 创建自定义 CORSFixMiddleware 直接添加响应头
5. ✅ 删除 Python __pycache__ 缓存
6. ✅ 使用 `--reload` 和不使用 `--reload` 启动
7. ✅ 修改后端监听地址为 `0.0.0.0:8000`

**问题现状：**
- 后端服务正常运行，直接访问接口返回数据正常
- PowerShell 测试请求成功，但响应头中仍无 CORS 头
- 中间件代码已添加，但未生效

**可能原因：**
1. PyCharm 或 uvicorn 缓存问题
2. Python 导入缓存未清除
3. 中间件执行顺序问题
4. FastAPI 版本兼容性问题

**待尝试方案：**
1. 在非 PyCharm 环境（普通终端）启动后端
2. 检查 FastAPI 和 Starlette 版本
3. 使用 `@app.middleware("http")` 装饰器方式添加 CORS
4. 检查是否有其他中间件冲突
5. 尝试降级或升级 FastAPI 版本
